Parmi tous les classes que nous avons conçues, la classe Controller rencontrait le contexte problématique du patron singleton. En effet, une seule instance de cette classe est nécessaire pour l'application et plusieurs classes référencient cette même instance de la classe contrôleur. Nous avons donc fait le nécessaire pour intégrer le patron singleton à la classe Controller. La cohésion est augmentée puisque que le client délègue la responsabilité de gestion des instances à la classe Controller.
